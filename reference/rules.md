# ルール詳細 — 設計 (D) / 実装 (I)

Fusion 360 Python スクリプトで「変化に強い・一意に決まる・編集しやすい」履歴付き部品を
作るための規範。`SKILL.md` の要約を補足する。

---

## 設計ルール (D)

### D0. マスター(レイアウト)スケッチを「設計の単一ソース」にする ★最上位
**部品の駆動ジオメトリ(skeleton)を 1 枚のマスタースケッチに集約**し、他スケッチ/フィーチャは
そこから projection/参照する。理由: スケッチが散ると **1 枚ずつ開いて確認・調整**が必要になり、
**全体を見回して一括調整できない**。マスターを見れば全体を把握・変更できる状態にする。

**「主要な情報」= マスターに入れる駆動ジオメトリ(= これを定義する。"全部" ではない):**
1. (必須) 部品の主平面フットプリント(外形・輪郭)。
2. (必須) **全フィーチャの位置基準**: 穴・ボス・リブ・ノッチ等の **中心点 / 基準線 / 軸**。
3. (必須) それらを関係づける **主要寸法・角度** → ユーザーパラメータにリンク。
4. (任意) 他平面スケッチが projection する **構築ジオメトリ**(中央寄せ矩形 等)。

**マスターに入れない(ローカルで可):**
- 別平面固有の "断面形" そのもの(位置は master から projection、半径等の形だけローカル+パラメータ)。
- 純粋にローカル完結の微細形状(面取り等)。

**なぜ "全部 1 スケッチ" にしないか**: スケッチは 1 平面に束縛される(多平面の形を 1 枚には描けない)。
よって正解は「**駆動ジオメトリ(主に X/Y skeleton)を master に集約 + 高さ等はパラメータ + 別平面は
projection**」。**設計の単一ソース = マスタースケッチ(skeleton)+ ユーザーパラメータ表**。この 2 つを
見れば、各スケッチを開かずに全体を調整できる。

> D1(1平面=1スケッチ)・D2(projection)・D6(対称の単一ソース)は、すべて **D0 を実現する手段**。

### D1. 1 平面 = 1 スケッチ(厳守)
**同一平面の全形状(外形・穴・切り欠き・繰り返し要素・seed)は 1 枚のスケッチに描く。**
押し出し/カットは複数あってよい(1 スケッチから**必要プロファイルを選択**して各フィーチャを作る)。
**スケッチを機能ごとに分けない。**
- 板に開く穴・縁の切り欠きは、**同じスケッチの内ループ/縁切り**として描く。外形押し出しは穴を除いた
  領域(=最大面積プロファイル, → I7)を選べば、板+穴+切り欠きが 1 操作で完成。
- 「base 板」と「フランジ」のように接した同一平面・同一押し出し量の形状は **1 つの外形**として描く
  (例: base 69×60.5 + 上下フランジ 8.5 → 実体は 1 枚の 69×77.5 矩形)。
- 同一平面で別操作(板=Cut の穴 と ボス=Join)が必要でも、**形状は 1 スケッチに集約**し、押し出しごとに
  プロファイルを選択する(→ cookbook「集約スケッチからのプロファイル選択」)。
- ❌ 悪い例: 穴用・ノッチ用・ボス用・リブ用に **seed スケッチを乱立**させ、同一平面に複数スケッチ。

### D1×D6. 対称形状を 1 スケッチに収めて両立させる(重要)
D6(対称はミラー/パターン)を D1 と両立させる手順。**seed のためにスケッチを増やさない**こと:
1. **穴/切り欠き(プレート内)** → 集約スケッチに**内ループ/縁切り**として全数を描き、外形押し出しで一括。
   対称性は ①原点中心の構築ジオメトリ(中央寄せ矩形の四隅 等)で位置を与える ②または `addSymmetry`
   拘束で関連付ける。→ 1 スケッチ・1 押し出しで完了。
2. **別平面の繰り返し(ボス/下穴 等)** → その平面の **1 スケッチに全数を描く**(位置は D2 の projection で
   master の対称点群から取得)。**全プロファイルをまとめて 1 押し出し**。
3. **どうしてもフィーチャパターン/ミラーを使う場合** → seed プロファイルを**集約スケッチから選択**して
   押し出し、その**フィーチャ**をパターン/ミラーする(新スケッチを作らない)。フィーチャパターン/ミラーが
   正当なのは「別平面・別ボディの繰り返し」で、かつ seed が既存スケッチ由来のとき。
> 結論: D6 の目的は「**対称を単一ソースから駆動**」。それは構築ジオメトリ+projection や addSymmetry で
> 達成でき、**スケッチ数を増やさずに**実現するのが D1 と整合する正解。

### D2. 平行平面へは master スケッチから projection
複数の平行平面(例: 底面 / 嵌合面 / 上面)に同じ位置の形状がある場合、**主要点・線・円は
1 つの master スケッチに描き**、他平面のスケッチへ `project()` して使う。
- 各平面でゼロから座標を打ち直さない(= 単一ソース。master を直せば全平面が追従)。
- 投影した点/線に対し、円や輪郭を **同心・一致拘束**で乗せる(→ I8)。
- 例: 取付穴中心を master(z=0)に置き、standoff(z=t)と下穴(z=t+h)の両スケッチへ projection。

### D3. 寸法を付与し、主要値はパラメータにリンク
スケッチには **寸法**(距離・直径・角度)を入れ、設計を数値で確定させる。
- 主要寸法は **ユーザーパラメータ**にリンク(`dim.parameter.expression = 'HAT_L'` 等)。
  → "パラメータを変更"ダイアログで一括編集 → 再構築。
- 押し出し距離・開始オフセットもパラメータ式で駆動(→ I4/I5)。

### D4. 拘束で変化に追従させる
座標ベタ置きでなく **幾何拘束**で関係を表現し、寸法変更時に形が破綻しないようにする。
- 使う拘束: coincident(一致)/ horizontal・vertical(水平垂直)/ parallel・perpendicular /
  equal(等しい)/ symmetric(対称)/ concentric(同心)/ tangent / collinear / midpoint。
- projection した参照に対しても coincident/concentric で結ぶ。

### D5. スケッチはフル拘束(一意に確定)
スケッチは **自由度 0(fully constrained)** にする。Fusion で線が**黒**になる状態。
- 「適切な拘束 + 寸法 + 角度」で形・位置・向きが一意に決まるようにする。
- 未拘束(青/白の自由な線)を残さない(後で勝手に動く・編集が壊れる原因)。
- 原点・対称面を基準に固定(原点に対する symmetric / midpoint で中心を固定すると安定)。

### D6. 対称形状は単一ソースから駆動(個別にバラ描きしない)
±対称の穴・ボス・リブは **独立にバラバラの数値で描かない**。対称を**1つの基準から**駆動する。
**ただし D1(1平面=1スケッチ)を壊さないこと**(→ 上の「D1×D6」が優先手順)。
- 推奨(D1 と整合): ①原点中心の**構築ジオメトリ**(中央寄せ矩形の四隅=対称な4点)で位置を与える /
  ②**projection** で master の対称点群を別平面へ流用 / ③スケッチ内 **addSymmetry** 拘束。
- フィーチャ **mirror / 矩形パターン**は「**別平面・別ボディの繰り返し**」かつ seed が集約スケッチ由来の
  ときに限り使用(seed 用の新スケッチを作らない)。
- 利点: ピッチを変えても対称性が自動で保たれる(D4 と一体)。
- ❌ アンチパターン: 「4 穴のために 4 つ円を別々の数値で描く」「対称要素ごとに seed スケッチを増やす」。
- レシピは cookbook の「対称 = 構築ジオメトリ/projection/addSymmetry」「集約スケッチからのプロファイル選択」。

## 実装ルール (I) — Fusion API の落とし穴

### I1. ルートコンポーネントに直接モデリング
Part デザインドキュメントは **1 コンポーネント制約**。`occurrences.addNewComponent(...)` は
`RuntimeError: パーツ デザイン ドキュメントには 1 つのコンポーネントのみ…` で失敗する。
→ サブコンポーネントを作らず `comp = design.rootComponent` に直接作る。

### I2. パラメトリック(タイムライン)を有効化
`design.designType = adsk.fusion.DesignTypes.ParametricDesignType` を最初に設定。
(Direct モデルだと履歴が残らない。)

### I3. 単位 = 内部 cm
Fusion API のジオメトリ内部単位は **cm**。mm 設計なら **作図座標は ×0.1**(`MM=0.1`)。
- 寸法パラメータは **mm 文字列**で定義: `ValueInput.createByString('65 mm')`(表示も mm)。

### I4. パラメータリンク = createByString(式)
押し出し距離・寸法値はパラメータ名/式で駆動: `ValueInput.createByString('BASE_T')`,
`createByString('BASE_T + STANDOFF_H')`。→ 後から一括編集できる。

### I5. ★別平面のスケッチは「押し出した面」に直接(オフセット平面を作らない)
次のフィーチャは、**前フィーチャの先端面 (`feat.endFaces`) に直接スケッチ**する
(`comp.sketches.add(face)`)。実ジオメトリに追従し(寸法変更で面が動けばスケッチも追従)、
構築平面が乱立せず、人手の操作に近い。projection はその面スケッチ上でそのまま機能し、
**先端面が複数(standoff 天面×4 等)でも 1 面に sketch + projection で全位置をカバー**できる。
- 取得: `feat.endFaces`(押し出し先端面)/ 無ければ `body.faces` を法線・z・面積で絞る。
- フォールバック(適切な面が無い初期段階のみ): オフセット構築平面 + `startExtent` =
  `OffsetStartDefinition.create(by('BASE_T'))`。**まず面スケッチを試し、面が無いときだけ平面**。
- cookbook「2. 別平面へのスケッチ = 押し出した面に直接」参照。

### I6. Join / Cut のターゲット明示
`extInput.participantBodies = [body]` を設定(対象ボディを明示)。Cut/Join の取りこぼし防止。

### I7. 板は最大面積プロファイルを選ぶ
穴/切り欠きを含むスケッチは複数プロファイルに分かれる。板本体は
`profile.areaProperties(LowCalculationAccuracy).area` が最大のものを選んで押し出す。

### I8. projection
`projected = target_sketch.project(master_point)` → `ppt = projected.item(0)`。
投影点に同心円を描き `target_sketch.geometricConstraints.addCoincident(circle.centerSketchPoint, ppt)`。

### I9. 寸法/拘束/projection は try/except で保護
API バージョン差で `addDistanceDimension` / `addDiameterDimension` / `project` / `addCoincident` /
`addSymmetry` の引数差・失敗があり得る。**各々 try/except で囲み、失敗しても本体形状は生成**する。

### I10. スクリプト配布形式
Fusion の「スクリプトとアドイン」は **フォルダ構造**必須:
```
<name>/
  <name>.py
  <name>.manifest   # {"autodeskProduct":"Fusion360","type":"script","supportedOS":"windows|mac","editEnabled":true,...}
```
フォルダ名 = .py 名 = .manifest 名 が一致。zip はこのフォルダごと固める。

## チェックリスト (生成前/後)
- [ ] **★平面ごとのスケッチ数 == 1**(超えていたら D1 違反 → 統合する。最重要点検)。
- [ ] seed のためにスケッチを増やしていない(集約スケッチからプロファイル選択している)。
- [ ] 平面ごとに形状を分類した(同一平面=1スケッチ統合 / 平行平面=projection)。
- [ ] 主要点/線は master 1 枚に集約し projection で再利用。
- [ ] 対称形状は構築ジオメトリ/projection/addSymmetry で単一ソース駆動(バラ描き・seed乱立していない)。
- [ ] 各スケッチがフル拘束(自由度0)。
- [ ] 主要寸法をユーザーパラメータにリンク。
- [ ] rootComponent 直接 / ParametricDesignType / 単位 mm×0.1。
- [ ] 寸法・拘束・projection を try/except で保護。
- [ ] 配布フォルダ構造 + manifest。
